What is Hidden Lineage? +

Hidden Lineage is a research-grade genetic archaeology project that analyzes modern DNA against 45,000 years of human history. Unlike commercial DNA tests that focus on recent ancestry, we use ancient genome databases to explore deep evolutionary connections.

Think of it as population archaeology applied to personal genetics - we're asking "where do your genes come from?" across archaeological time scales rather than genealogical ones.

How is this different from 23andMe or AncestryDNA? +

Commercial DNA tests and Hidden Lineage serve different purposes:

  • Commercial Tests: Find recent relatives, health insights, genealogy (~500 years)
  • Hidden Lineage: Explore ancient population connections, archaeological ancestry (~45,000 years)

We use different methods (ADMIXTURE vs IBD), different reference data (ancient + modern vs modern only), and answer different questions (population structure vs family trees).

Both approaches provide valuable but complementary insights about your ancestry.

Is this scientifically valid? +

Yes. Hidden Lineage uses the same computational methods found in peer-reviewed population genetics research:

  • ADMIXTURE algorithm (widely used in academic studies)
  • Allen Ancient DNA Resource (AADR) - standard reference database
  • Cross-validation for model selection
  • Transparent methodology with clear limitations

However, this is preliminary research based on chromosome 22 only. Full genome analysis will provide more robust results.

Validation: Our methods follow established protocols from Reich Lab (Harvard), Pickrell Lab, and other leading population genetics groups. All software versions, parameters, and data sources are documented for reproducibility.

What does "Chr22 only" mean? +

Currently, our analysis focuses on chromosome 22 for computational efficiency and rapid prototyping. This means:

  • Advantages: Faster analysis, proof of concept, real data insights
  • Limitations: Reduced statistical power, potential chromosome-specific biases
  • Future: Full genome analysis (all 22 autosomes) planned within 4-6 weeks

Think of current results as a "preview" - directionally accurate but will be refined with complete genomic data.

What is ADMIXTURE and how does it work? +

ADMIXTURE is a computational tool that models your genome as a mixture of ancestral populations. It uses maximum likelihood estimation to determine:

  • How many ancestral components (K) best explain genetic variation
  • What proportion of your genome comes from each component
  • Which reference populations each component represents

Think of it like unmixing paint colors - if your genome is purple, ADMIXTURE figures out how much "blue ancestry" and "red ancestry" went into creating it.

Algorithm: Uses expectation-maximization (EM) algorithm with block relaxation for optimization. Each iteration improves the likelihood estimate until convergence (delta < 0.0001).

K Selection: Cross-validation determines optimal number of ancestral components, preventing overfitting while maximizing explanatory power.

Why K=27? What does that number mean? +

K=27 means the analysis identified 27 distinct ancestral components that best explain genetic variation in our dataset. This number comes from:

  • Cross-validation: K=27 achieved the lowest CV error (0.63218)
  • Objective selection: Algorithm chose this, not human interpretation
  • Still improving: K=28+ might be even better (analysis ongoing)

Higher K values can reveal finer population structure but may also introduce artifacts from reference panel imbalances.

CV Trend: Error decreased from 0.65126 (K=5) to 0.63218 (K=27), suggesting optimal K hasn't been reached yet.

Comparison: Most commercial tests use K=5-7, academic studies often explore K=10-50+ depending on dataset size and diversity.

What are "components" and do they represent real populations? +

Components are statistical clusters, not literal populations. They represent:

  • Genetic signatures: Patterns of shared ancestry
  • Population proxies: Named after highest-loading reference samples
  • Latent structure: Hidden patterns in genetic data

For example, "Component 10: Armenia Urartian IA" means this component is strongest in ancient Armenian samples, but it represents a broader genetic signature that may extend beyond that specific population.

Mathematical Definition: Each component is a frequency vector across all SNPs, representing an idealized ancestral population's allele frequencies.

Interpretation Caution: High K values may split meaningful populations or create artifacts. Component stability across K values helps validate biological relevance.

How reliable are these results? +

Results have varying levels of reliability:

  • High confidence: Major component trends, broad population affinities
  • Medium confidence: Specific component percentages, population labels
  • Low confidence: Minor components (<5%), exact historical connections

Current Chr22-only analysis provides directional insights that will be validated and refined with full genome data.

Error Sources: Linkage disequilibrium on single chromosome, reference panel imbalances, ancient DNA degradation, algorithm convergence.

Validation Plan: Bootstrap resampling, component stability testing, full genome replication, independent dataset validation.

The differences reflect different analytical approaches:

Commercial tests answer "who are your recent relatives?" while Hidden Lineage asks "what ancient populations contributed to your genetic makeup?"

What does it mean to have "Armenia Urartian IA" ancestry? +

This means your genome shares genetic patterns with ancient populations from the Armenian highlands during the Iron Age (roughly 1200-600 BCE).

Important caveats:

  • This doesn't mean direct descent from those specific people
  • The genetic signature may have spread through migration, trade, or population mixing
  • Component labels reflect highest-loading samples, not exclusive origins

Think of it as genetic evidence of ancient population connections across archaeological time scales.

Can these results tell me about recent family history? +

Not directly. Hidden Lineage focuses on population-level patterns rather than individual genealogy:

  • What it shows: Deep ancestral population connections, migration patterns, archaeological heritage
  • What it doesn't show: Recent relatives, specific family trees, genealogical connections

For recent family history, commercial DNA tests or genealogical research are more appropriate tools.

Why do some components have unusual population combinations? +

Unusual combinations like "Han Chinese + Ukraine EBA" can result from:

  • Ancient migrations: Populations moved and mixed over millennia
  • Shared ancestry: Distant common origins before population splits
  • Reference artifacts: Imbalanced sampling or high K effects
  • Complex history: Trade routes, conquests, or cultural exchanges

These patterns will be clarified through full genome analysis and reference panel improvements.

Why only chromosome 22? How does this affect results? +

Chromosome 22 was chosen for computational efficiency and rapid prototyping, but this creates important limitations:

  • Reduced statistical power: ~2% of genome vs 100%
  • Potential chromosome bias: Chr22 may not represent genome-wide patterns
  • Lower resolution: Fewer SNPs mean less precise ancestry estimates
  • Missing signals: Some population-specific variants may be absent

Current results should be viewed as directional insights that will be validated with full genome analysis.

Technical impact: Chr22 contains ~7,400 SNPs vs ~600,000+ genome-wide. This 100x reduction in data affects both resolution and confidence intervals significantly.

What ancient populations are in your reference panel? +

Our reference panel includes ~15,000 ancient genomes spanning 45,000 years:

  • Paleolithic: Hunter-gatherers (45,000-10,000 BCE)
  • Neolithic: Early farmers (10,000-3,000 BCE)
  • Bronze Age: Metallurgists and pastoralists (3,000-1,200 BCE)
  • Iron Age: Complex societies (1,200-0 CE)
  • Historical: Medieval and recent populations (0-1,500 CE)

Geographic coverage includes Europe, Asia, Africa, and the Americas, though with uneven sampling density.

Data sources: Allen Ancient DNA Resource (AADR v62), supplemented by recent publications from Reich Lab, Max Planck Institute, and collaborating ancient DNA laboratories worldwide.

How do you handle contamination in ancient DNA? +

Ancient DNA quality control is critical for accurate results:

  • Contamination filters: Only samples with <5% modern contamination
  • UDG treatment: Preference for UDG-treated libraries (reduces damage artifacts)
  • Pseudo-haploid calls: Random allele sampling to avoid reference bias
  • Coverage thresholds: Minimum SNP coverage requirements per sample

Quality metrics: AADR samples include contamination estimates from multiple methods (mtDNA, X-chromosome, ANGSD). We exclude high-contamination samples and prefer those with authentication evidence.

What's the oldest sample in your dataset? +

The oldest high-quality samples date to approximately 45,000 years ago:

  • Ust'-Ishim (Siberia): ~45,000 years old
  • Oase 1 (Romania): ~40,000 years old
  • Bacho Kiro (Bulgaria): ~45,000 years old
  • Tianyuan (China): ~40,000 years old

These Paleolithic samples provide crucial insights into early human migrations and population structure before major demographic transitions.

Technical note: Very ancient samples often have low coverage and high damage rates. We include them for population structure analysis but weight them appropriately in statistical models.

How do you choose the optimal K value? +

K selection uses cross-validation to balance model complexity with explanatory power:

  • Cross-validation error: 5-fold CV with lowest error indicating best K
  • Diminishing returns: Error reduction plateaus suggest optimal range
  • Biological interpretation: Components should reflect known population structure
  • Stability testing: Consistent results across multiple runs

K=27 achieved the lowest CV error (0.63218) in our current analysis, but K=30+ may perform even better.

Algorithm details: ADMIXTURE uses 5-fold cross-validation with random partitioning. Each K value runs 5 replicates with different seeds, selecting the best likelihood result for CV calculation.

Why do I have Central Asian ancestry if my family is European? +

Unexpected ancestry components can reflect deep historical processes:

  • Ancient migrations: Steppe pastoralists spread into Europe ~5,000 years ago
  • Shared deep ancestry: Common origins before population divergence
  • Trade and exchange: Silk Road and other contact networks
  • Statistical artifacts: High K values may create spurious components

Modern populations are mixtures of many ancient sources - geography doesn't always predict genetic ancestry.

Example: The "Yamnaya" component in Europeans reflects Bronze Age migrations from the Pontic Steppe. This creates statistical similarity with Central Asian populations who share this ancestry.

How certain are these percentages? +

Ancestry percentages have varying levels of certainty:

  • Major components (>10%): Relatively stable across runs
  • Medium components (2-10%): Moderate confidence, may fluctuate
  • Minor components (<2%): Low confidence, possibly noise
  • Chr22 limitation: All estimates less precise than genome-wide

Focus on broad patterns rather than exact numbers. Full genome analysis will provide confidence intervals.

Technical note: ADMIXTURE doesn't provide confidence intervals by default. Bootstrap resampling and multiple K comparisons help assess stability, but formal uncertainty quantification requires additional analysis.

Do these results mean I'm descended from specific historical groups? +

Not directly. ADMIXTURE components represent statistical similarities, not genealogical descent:

  • Population-level patterns: Shared ancestry at demographic scales
  • Not individual lineages: These aren't family trees or direct descent
  • Complex mixing: Multiple sources contribute to each component
  • Time averaging: Components may reflect thousands of years of mixing

Think of results as "genetic signatures" that connect you to ancient population movements, not specific tribal affiliations.

Why are some components so small (<1%)? +

Very small components may reflect either real ancestry or statistical noise:

  • Real but ancient: Deep ancestral connections diluted over time
  • Reference panel artifacts: Overrepresented populations create spurious matches
  • High K effects: Complex models can generate artificial components
  • Chr22 sampling: Limited data may amplify noise

Generally ignore components <1% until validated with full genome analysis and balanced panels.

Statistical threshold: Most population geneticists consider components <2-3% potentially artifactual, especially at high K values. Component stability across different K helps distinguish signal from noise.

How does this compare to academic papers using ADMIXTURE? +

Hidden Lineage follows the same methodological framework as published research:

  • Same software: ADMIXTURE v1.3.0, widely used in population genetics
  • Same data sources: AADR and 1000 Genomes (standard references)
  • Same quality filters: MAF, missingness, LD pruning protocols
  • Academic precedent: K=20-50 commonly used in ancient DNA studies

Key differences: single chromosome (computational limitation) and individual focus (vs population comparisons).

Literature comparison: Similar approaches in Haak et al. 2015 (Nature), Lazaridis et al. 2016 (Nature), and dozens of Reich Lab publications. Our parameters and preprocessing match established protocols.

Why don't you show confidence intervals? +

ADMIXTURE doesn't provide confidence intervals by default, but we're working on uncertainty quantification:

  • Technical limitation: Standard ADMIXTURE output lacks error estimates
  • Bootstrap planned: Resampling methods to estimate variance
  • Cross-K comparison: Stability across different K values as proxy
  • Full genome priority: Confidence intervals more meaningful with complete data

Methods under development: Block bootstrap resampling, jackknife validation, and Bayesian approaches (e.g., STRUCTURE) that provide posterior probability distributions.

Will you add Neanderthal and Denisovan ancestry? +

  • Phase 3 feature: Planned after core ADMIXTURE pipeline is complete
  • Interesting insights: Could reveal regional patterns of archaic ancestry

For now, focus remains on modern human population structure, but archaic ancestry is definitely on the roadmap.

Technical approach: Will likely use ADMIXTOOLS suite with qpWave/qpAdm for formal tests of archaic introgression, complementing the ADMIXTURE-based population structure analysis.

Why doesn't 23andMe show ancient ancestry? +

Commercial tests are optimized for different goals:

  • Business focus: Health, genealogy, recent relatives drive customer value
  • Technical approach: IBD segment matching works best for recent timeframes

Which approach is more accurate? +

Accuracy depends on what you're trying to measure:

  • For recent ancestry (< 500 years): Commercial IBD methods are more accurate
  • For ancient ancestry (> 1000 years): ADMIXTURE with ancient references is more informative
  • For health/medical: Commercial tests use validated, FDA-approved markers
  • For population history: Research methods provide deeper insights

Think of them as different tools for different jobs, each accurate within their intended domain.

What file formats do you support? +

Currently processing 23andMe raw data format, but the pipeline supports:

  • Input: 23andMe, AncestryDNA, MyHeritage raw data files
  • Processing: PLINK binary format (.bed/.bim/.fam)
  • Output: ADMIXTURE results (.Q/.P), visualizations, reports

Preprocessing: All formats converted to GRCh37 coordinates, filtered for quality (MAF > 0.01, missing < 10%), LD-pruned (r¬≤ < 0.2), and merged with reference panels using PLINK v1.9.
