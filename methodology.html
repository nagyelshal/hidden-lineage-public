Research Methodology

How we analyze modern DNA against 45,000 years of human history

🔬 Analysis Pipeline Overview

Hidden Lineage uses research-grade population genetics tools to compare modern DNA against comprehensive ancient genome databases. This methodology follows established academic protocols while making them accessible for personal ancestry analysis.

1
Data Preprocessing
Raw DNA file converted to research format (PLINK), filtered for quality, and standardized to GRCh37 coordinates
2
Reference Panel Assembly
16,770 ancient and modern samples merged from AADR, SGDP, and 1000 Genomes databases
3
SNP Intersection
Identify overlapping genetic markers between your DNA and reference panels (7,401 sites on Chr22)
4
ADMIXTURE Analysis
Maximum likelihood clustering to infer ancestral components across K=3 to K=27+ values (note: high K may introduce panel imbalance artifacts)
5
Cross-Validation
5-fold CV to determine optimal K value and validate model stability
6
Population Inference
Components interpreted based on highest-loading reference individuals and archaeological context

🧪 Technical Implementation Details

Quality Control: SNPs filtered for MAF > 0.01, missing data < 10%, HWE p > 1e-6

Linkage Pruning: LD-based pruning using --indep-pairwise 50 5 0.2 (50 SNP window, 5 SNP step, r² < 0.2)

ADMIXTURE Convergence: Multiple replicates per K with different random seeds, best replicate selected by highest log-likelihood

Ancient DNA Treatment: Pseudo-haploid calls only, UDG-treated samples preferred, contamination < 5%

Computational Resources: Analysis performed on Ubuntu 22.04 LTS with 64GB RAM, 4-core parallel processing

🧪 ADMIXTURE Algorithm Details

# File: aadr_HO_chr22_filtered.{bed, bim, fam} # Multiple replicates with different seeds for seed in {1..5}; do admixture --cv=5 -j4 -s $seed aadr_HO_chr22_filtered.bed K done # Best replicate selected by highest log-likelihood

Maximum Likelihood Estimation: Uses expectation-maximization algorithm to estimate ancestry proportions

Model Selection: Cross-validation error guides optimal K selection, not visual interpretation

Convergence Criteria: Log-likelihood delta < 0.0001 between iterations

Replicate Selection: 5 replicates per K value, best likelihood score retained

📚 Reference Datasets

Allen Ancient DNA Resource (AADR v62)

Comprehensive database of ancient genomes spanning 45,000 years of human history. Includes Paleolithic hunter-gatherers, Neolithic farmers, Bronze Age populations, and historical samples through medieval periods.

Simons Genome Diversity Project (SGDP)

High-coverage modern genomes representing global population diversity. Provides contemporary baseline for comparing ancient signals.

1000 Genomes Project Phase 3

Large-scale modern human variation dataset covering major continental populations. Essential for understanding recent demographic history.

🧪 Sample Composition Breakdown

Ancient DNA (AADR): ~15,000 samples from archaeological sites worldwide

Ancient DNA Filters: UDG-treated preferred, contamination estimates < 5%, pseudo-haploid calls only

Modern High-Coverage (SGDP): 279 individuals from 142 populations

Population Genetics (1000G): 2,504 individuals from 26 populations

Geographic Coverage: All inhabited continents, 45,000 year time span

Data Format: All samples converted to pseudo-haploid format for consistency

⚠️ Current Limitations & Future Improvements

Current Limitations

  • Chromosome 22 Only: Single chromosome analysis reduces power and may miss genome-wide patterns
  • Reference Panel Imbalance: Some populations over-represented, leading to potential assignment artifacts at high K
  • No IBD Analysis: Focus on population structure rather than recent genealogical connections
  • Preliminary Results: Chr22-based estimates will be refined with full genome analysis
  • Population Labels: Component assignments based on reference individuals, not direct ancestry claims

Planned Improvements

  • Full Genome Analysis: Expand to all 22 autosomes for comprehensive ancestry inference
  • Panel Rebalancing: Add underrepresented populations and downsample over-represented groups
  • Rare Dataset Integration: Include specialized ancient DNA collections for specific regions
  • IBD Complement: Add identity-by-descent analysis for recent genealogical connections
  • Uncertainty Quantification: Bootstrap confidence intervals for ancestry estimates
  • Interactive Visualization: Dynamic exploration of results across different K values

🧪 Validation Strategy

Cross-Validation: 5-fold CV ensures model generalizability and prevents overfitting

K Selection: Objective CV error minimization rather than subjective interpretation

Reference Validation: Known population relationships verified in analysis

Reproducibility: Standardized pipeline with version-controlled dependencies

Quality Assurance: Pipeline frozen at Git commit hash for reproducible results

Statistical Significance: Bootstrap resampling planned for confidence intervals in full genome analysis

🔄 How This Differs from Commercial DNA Testing

Commercial DNA Tests

  • Reference Data Modern only
  • Time Depth ~500 years avg*
  • Method IBD segments
  • Categories 5-7 broad
  • Purpose Genealogy

Hidden Lineage

  • Reference Data Ancient + Modern
  • Time Depth 45,000 years
  • Method ADMIXTURE
  • Components 27+ detailed
  • Purpose Archaeology

*Average genealogical time depth; some IBD segments can extend to ~1,000 years. Both approaches provide valuable but different insights. Commercial tests excel at recent genealogy and health-related variants, while Hidden Lineage explores deep evolutionary history and archaeological population connections.

🧪 Methodological Complementarity

IBD vs ADMIXTURE: Different algorithms answer different questions about ancestry

Time Scales: Recent vs deep evolutionary history serve different research purposes

Reference Panels: Modern convenience samples vs archaeological specimens

Interpretation: Genealogical relationships vs population structure
